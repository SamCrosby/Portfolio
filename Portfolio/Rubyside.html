Rubyside

The Problem

The problem was that the Rubyside website was starting to look a little dated and so needed redesigning to bring it up to date with modern web page user interface trends. The goal of the website was to get users to sign up to the event.

Design

Low-fidelity prototypes were used to create quick, rough sketches of different designs. High-fidelity prototypes were used to create the final look of the website. The website needed to include information about the next event, rubyside, the organisers, cleversteam, whilst also including a link to the Rubyside twitter and Slack channel.

Build

The website was created using HTML and CSS. This build is split into three main sections in the body, a what is rubyside section, a next event section and an organisers section. The next event section includes information on the next event, as well as a button that takes the user to eventbrite to register for the event. The organisers section includes the organisers faces in rubys as a fun way to represent the ruby brand.

Test

This build was tested using manual testing, which involved creating a test plan containing a number of test cases. When this test was conducted, the result was written up, along with whether or not the build passed this test.
